# Auth & Security

## 1) OAuth 2.0 cho Google Workspace

Agent sử dụng OAuth 2.0 để user đăng nhập và cấp quyền truy cập Google Workspace.

Nguyên tắc:

- Mỗi account user có token riêng, không dùng chung.
- Luôn xin quyền theo **least privilege**.
- Tách rõ scope read và write để giảm blast radius.

Scope MVP đề xuất:

- Gmail: read, compose, send.
- Google Calendar: read/create/update events.
- Google Docs: read/create/update documents.
- Drive: read metadata/file, upload/update file.
- Sheets: read/write range.

## 2) Token & Credential Management

- Access token lưu tạm thời, refresh token mã hóa khi lưu local/backend.
- Dùng secret store phù hợp theo OS (Keychain/Credential Manager/libsecret) nếu lưu phía desktop.
- Rotate/revoke token khi user logout hoặc nghi ngờ compromise.

## 3) Authorization Policy Layer

Trước khi execute action, hệ thống kiểm tra:

- User đã cấp đúng scope chưa.
- Action có nằm trong policy cho phép không.
- Action có yêu cầu HITL bắt buộc không.

Ví dụ policy:

- `read_email`: low risk, có thể auto-run.
- `create_draft`: medium risk, cho phép auto-run hoặc HITL theo setting.
- `send_email`: high risk, HITL bắt buộc.

## 4) HITL as Security Gate

HITL không chỉ là UX, mà là lớp bảo mật nghiệp vụ:

- Chặn hành động nhạy cảm trước khi gọi API thật.
- Cho phép user chỉnh sửa nội dung trước khi gửi.
- Tạo bằng chứng phê duyệt cho audit/compliance.

## 5) Audit Logging

Ghi log đầy đủ cho chuỗi hành động:

- `task_id`, `action_id`, `user_id`, timestamp.
- Input đã chuẩn hóa (đã masking dữ liệu nhạy cảm nếu cần).
- Trạng thái approval: pending/approved/rejected/edited.
- Kết quả execute: success/failure + error code.

## 6) Runtime Safety Controls

- Rate limit theo user và theo tool để tránh spam/quota abuse.
- Retry có giới hạn cho lỗi tạm thời, không retry vô hạn.
- Timeout cứng cho action call.
- Idempotency key cho hành động có khả năng bị gửi lặp.

## 7) Data Protection

- Mã hóa dữ liệu nhạy cảm at-rest và in-transit.
- Không log raw email body/file content nếu không cần thiết.
- Cấu hình data retention rõ ràng (x ngày) cho log và lịch sử.
