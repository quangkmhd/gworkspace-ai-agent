# Task Breakdown

## Phase 1 - Foundation

- T1: Setup monorepo structure (`app`, `backend`, `agent`, `tools`, `hitl`).
- T2: Define shared action schema và risk-level mapping.
- T3: Thiết lập logging, tracing, error format thống nhất.

**Deliverable:** Project skeleton chạy được local với health check.

## Phase 2 - Auth & Workspace Integration

- T4: Implement Google OAuth 2.0 login flow.
- T5: Token storage + refresh + revoke flow.
- T6: Kết nối Gmail API (read, draft, send).
- T7: Kết nối Google Calendar API (read/create/update events).
- T8: Kết nối Google Docs API (read/create/update documents).
- T9: Kết nối Drive API (list, upload, share).
- T10: Kết nối Sheets API (read/write ranges).

**Deliverable:** Có thể gọi từng tool độc lập bằng API nội bộ.

## Phase 3 - Agent Core

- T11: Xây planner nhận prompt -> tạo plan nhiều bước.
- T12: Xây executor map plan -> tool actions.
- T13: Implement risk evaluator (Low/Medium/High).
- T14: Kết nối planner/executor với HITL engine.

**Deliverable:** End-to-end từ prompt đến action proposal.

## Phase 4 - HITL Core (differentiator)

- T15: Implement approval queue + state machine.
- T16: API cho approve/edit/reject.
- T17: Desktop approval panel + action preview.
- T18: Native notification khi có pending approval.
- T19: Audit log cho toàn bộ vòng đời phê duyệt.

**Deliverable:** User duyệt action trực tiếp trên desktop app trước khi execute.

## Phase 5 - Desktop UX

- T20: Chat timeline hiển thị plan/action/result rõ ràng.
- T21: History view cho task và approvals.
- T22: Settings cho policy cơ bản (auto-approve low risk, timeout).

**Deliverable:** Desktop app usable cho daily workflow cơ bản.

## Phase 6 - Hardening & Launch Prep

- T23: Unit/integration tests cho agent/tools/hitl.
- T24: E2E scenario: "read sheet -> draft -> approve -> send email".
- T25: E2E scenario: "tao su kien Calendar -> canh bao xung dot -> phe duyet".
- T26: E2E scenario: "tao Docs tu template -> cap nhat noi dung -> chia se".
- T27: Security review (scope, token, audit, retry/idempotency).
- T28: Packaging và phát hành bản Win/Mac/Linux.

**Deliverable:** MVP release candidate.

## Phase 7 - Advanced (post-MVP)

- T29: Multi-agent mode (planner/action/reflection).
- T30: Memory dài hạn theo user/task context.
- T31: Auto-approve rules nâng cao theo domain/team.
- T32: Thêm app mới ngoài phạm vi hiện tại (Meet, Chat, Admin tools).
