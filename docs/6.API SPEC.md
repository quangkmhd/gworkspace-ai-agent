# API Spec (Agent-Ready)

## Mục tiêu

Thiết kế API đầy đủ để một agent khác có thể gọi tool trực tiếp qua endpoint, đồng thời vẫn đi qua HITL gate theo policy.

## Chuẩn chung

- Base URL: `/v1`
- Auth: `Authorization: Bearer <token>`
- Idempotency cho write/send/delete: `Idempotency-Key` header
- Response envelope:

```json
{
  "ok": true,
  "data": {},
  "error": null,
  "meta": {"request_id": "req_001"}
}
```

## 1) Agent Orchestration Endpoints

### 1.1 Create Task

`POST /v1/agent/tasks`

```json
{
  "user_id": "u_001",
  "prompt": "Tao lich hop tuan sau va gui email nhac team",
  "context": {"locale": "vi-VN"}
}
```

### 1.2 Get Task

`GET /v1/agent/tasks/{task_id}`

### 1.3 List Task Actions

`GET /v1/agent/tasks/{task_id}/actions`

### 1.4 Get Action Detail

`GET /v1/agent/actions/{action_id}`

## 2) Tool Registry & Invoke (quan trọng để tích hợp agent khác)

### 2.1 List Tools

`GET /v1/tools`

Trả về danh sách tool name, input schema, risk default, cần HITL hay không.

### 2.2 Get Tool Schema

`GET /v1/tools/{tool_name}`

Ví dụ `tool_name = gmail.send_email`.

### 2.3 Invoke Tool (generic)

`POST /v1/tools/{tool_name}/invoke`

```json
{
  "task_id": "tsk_001",
  "actor": "agent_sanic_01",
  "args": {},
  "dry_run": false
}
```

`dry_run=true` dùng để preview và kiểm tra policy/HITL trước khi execute thật.

## 7) Minimal tool invoke examples

### Gmail send (high risk => HITL required)

`POST /v1/tools/gmail.send_email/invoke`

```json
{
  "task_id": "tsk_001",
  "actor": "agent_sanic_01",
  "args": {
    "to": ["team@company.com"],
    "subject": "Weekly report",
    "body": "..."
  }
}
```

### Calendar move event (medium/high by policy)

`POST /v1/tools/calendar.move_event/invoke`

```json
{
  "task_id": "tsk_002",
  "actor": "agent_sanic_01",
  "args": {
    "event_id": "evt_001",
    "source_calendar_id": "primary",
    "target_calendar_id": "team_calendar"
  }
}
```
