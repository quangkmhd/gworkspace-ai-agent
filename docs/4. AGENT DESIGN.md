# Agent Design

## 1) Agent Loop (single-agent MVP)

```python
while True:
    intent = understand_user_input(user_input)
    # Lập kế hoạch theo cấu trúc và ưu tiên các Tool có sẵn trong LangChain Google Workspace
    plan = build_plan(intent, context, memory)
    actions = map_plan_to_tools(plan) # Ưu tiên LangChain Tools > Custom Adapters

    for action in actions:
        risk = evaluate_risk(action)
        if risk.requires_hitl:
            decision = wait_for_user_approval(action)
            if decision == "approve":
                execute(action)
            elif decision == "edit":
                execute(decision.edited_action)
            else:
                cancel(action)
        else:
            execute(action)

    summarize_result_to_user()
```

## 2) Action Model chuẩn hóa

## 3) Planning Strategy

- Ưu tiên kế hoạch ngắn, có thể kiểm chứng từng bước.
- Mỗi bước phải map được về tool cụ thể, ưu tiên tuyệt đối **LangChain Google Workspace Tools**.
- Chỉ dùng custom-defined tools (qua Google API trực tiếp) nếu LangChain chưa hỗ trợ tính năng (ví dụ: Google Docs format, file batch export nâng cao).
- Nếu thiếu dữ liệu quan trọng, agent phải hỏi user thay vì tự suy diễn.

Ví dụ decomposition:

1. Đọc dữ liệu từ Sheet.
2. Tóm tắt nội dung.
3. Tạo email draft.
4. Chờ user duyệt trước khi gửi.

## 4) Risk Evaluation

Risk evaluator dựa trên:

- Loại hành động (read/write/send/share/delete).
- Đối tượng tác động (nội bộ/ngoài tổ chức).
- Mức độ không thể hoàn tác.

Rule mặc định:

- **Low**: read-only, không HITL.
- **Medium**: thay đổi có thể hoàn tác, HITL optional.
- **High**: gửi/chia sẻ/xóa/ghi đè quan trọng, HITL bắt buộc.

## 5) Error Handling & Recovery

- Tool error: retry có kiểm soát và trả lỗi rõ ràng cho user.
- Approval timeout: action chuyển trạng thái expired.
- Partial failure trong plan nhiều bước: dừng chuỗi và báo trạng thái từng bước.

## 6) Multi-agent (phase nâng cao)

Khi mở rộng, có thể tách thành:

- **Planner Agent**: tạo kế hoạch.
- **Action Agent**: thực thi tools.
- **Reflection Agent**: kiểm tra chất lượng và đề xuất sửa.

Mục tiêu của multi-agent là tăng độ ổn định trong task phức tạp, không thay thế HITL gate.
